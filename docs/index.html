Why Not MCP? Solving Multi-Agent Distributed Scale

Model Context Protocol (MCP) excels at `1:1` client-server prompt attachment. ACP solves the foundational distributed challenges of `N:M` autonomous agent collaboration.

Architectural Dimension Model Context Protocol (MCP) Agent Collaboration Protocol (ACP)
Topological Model Client-Server (1:1 / 1:N)
Star topology where a single local IDE frontend controls context.		 Distributed Mesh (N:M)
Peer-to-peer, gossip cluster, and hierarchical topology with no single point of failure.
Interaction Pattern Synchronous RPC
Blocking tool calls (`tools/call`) and resource fetching.		 Async & Streaming First
Non-blocking multi-modal streaming (`STREAM_CHUNK`), DAG execution, and SLA negotiations.
Identity & Security Local Process / Implicit Trust
Relies on local `stdio` subprocess isolation or basic HTTP keys.		 Zero-Trust Cryptographic DIDs
W3C DIDs (`did:acp:...`), canonical Ed25519/HMAC frame signing, Capability Tokens, and mTLS 1.3.
Orchestration & Workflows None (Client Loop Scope)
Left entirely to the client application LLM execution loop.		 Distributed Workflow Engine
Parallel DAG tasks, SLA pricing (`PROPOSE->ACCEPT`), Voting Gates (`>= 66.7% consensus`), and reverse Saga Rollback upon failure.
Distributed Memory Stateless Resource Reads
Queries via `resources/read` without cross-node state synchronization.		 6-Tier Synchronized Fabric
Short-Term LRU, Long-Term KV, Vector Cosine RAG, KG Triples, and CRDT LWW-Element-Set conflict-free synchronization.
Network Transports stdio & HTTP/SSE
Primarily designed for local desktop process pipes and SSE.		 Transport Agnostic Wire Frame
HTTP/1/2/3, QUIC, gRPC, WebSocket, Unix Domain Sockets, TCP/UDP, and P2P Mesh (`libp2p`).

Universal Wire Frame Envelope

Every message across HTTP, gRPC, QUIC, or WebSocket is encapsulated inside the universal envelope schema (`draft-2020-12`), ensuring verifiable signatures and W3C trace propagation.


            

        

    


    
    

        

            
6-Tier Distributed Memory Fabric

            
Agents collaborating across machines require synchronized semantic state. ACP Memory partitions state across six specialized consistency tiers.

        

        

            

                
⚡

                
1. Short-Term LRU Cache

                SLA Latency: < 1 ms
                
High-speed working scratchpad for intermediate calculation states, active prompt buffers, and temporary session scratchpads. Auto-evicted on TTL or session end.

            

            

                
💾

                
2. Long-Term Key-Value Store

                SLA Latency: < 10 ms
                
Persistent, durable storage for user preferences, historical agent execution logs, and domain entities backed by RocksDB or distributed DynamoDB tables with RF=3 replication.

            

            

                
🔄

                
3. Shared CRDT Memory

                SLA Latency: < 5 ms
                
Lock-free collaborative workspace state across concurrent agents using LWW-Element-Set (Last-Write-Wins). Highest logical timestamp deterministically merges without distributed locking bottlenecks.

            

            

                
🌌

                
4. Vector Cosine RAG Index

                SLA Latency: < 25 ms
                
Dense high-dimensional embedding storage (HNSW graph index) supporting Cosine Similarity and L2 Euclidean queries with exact metadata tag filtering for instant semantic RAG retrieval.

            

            

                
🕸️

                
5. Knowledge Graph Triples

                SLA Latency: < 30 ms
                
Structured relational reasoning over entity networks (`Subject -> Predicate -> Object`). Supports multi-hop depth traversal (`max_depth=3`) to trace causal dependencies between AI agents and tools.

            

            

                
🔒

                
6. Encrypted Secure Enclave

                SLA Latency: < 15 ms
                
Data-at-rest AES-256-GCM encryption partitions for sensitive PII, healthcare records (`HIPAA`), or API keys. Accessible only by agents presenting a valid cryptographically signed Capability Token.

10 Production-Grade RFC Specifications

            
Every architectural decision includes complete technical reasoning, scalability math, security analysis, and exact wire formatting suitable for enterprise adoption.

        

        

            
                RFC-ACP-001
                
Architecture & Vision Specification

                
Executive summary, comparison with MCP, core principles (`Streaming-First`, `Zero-Trust`), and canonical HMAC/Ed25519 payload canonicalization.

            
            
                RFC-ACP-002
                
Components & Roles Deep-Dive

                
Complete technical specification of all 15 components: Host, Agent, Gateway, Router, Registry, Discovery, Memory Service, Scheduler, Broker, IdP, Marketplace...

            
            
                RFC-ACP-003
                
Communication & Transport Bindings

                
Semantics for 9 communication vectors (`Agent->Agent/Tool/Human/DB`), network transport adapters (HTTP3/gRPC/QUIC/WebSocket/P2P), and credit flow control.

            
            
                RFC-ACP-004
                
DAG Workflow & Orchestration Engine

                
Directed Acyclic Graph execution, SLA negotiation (`PROPOSE->COUNTER->ACCEPT`), Voting Gates (`>= 66.7% consensus`), checkpoints, and reverse Saga Rollbacks.

            
            
                RFC-ACP-005
                
Distributed Memory Architecture

                
Deep-dive into 6-Tier memory: Short-Term LRU, Long-Term KV, CRDT LWW-Element-Set synchronization, Cosine Vector RAG, KG Triples, and copy-on-write snapshots.

            
            
                RFC-ACP-006
                
Enterprise Zero-Trust Security

                
W3C DIDs (`did:acp:...`), OAuth2/OIDC Token Exchange (`RFC 8693`), mTLS 1.3, Capability Tokens, ABAC claims, Replay defense (`nonce + TTL`), and Secrets Vault.

            
            
                RFC-ACP-007
                
Discovery & Marketplace Catalog

                
Multi-tier discovery (`mDNS` + `SWIM Gossip` + `Registry Queries`), semantic vector skill indexing, and verified OCI package publishing pipeline.

            
            
                RFC-ACP-008
                
Runtime & OpenTelemetry Suite

                
Execution isolation across Docker, K8s Pods, Serverless, Bare Metal (`cgroups v2`), and Wasm (`WASI`), plus W3C traceparent propagation (`00-4bf92f35...-01`).

            
            
                RFC-ACP-009
                
Official 14-Language SDK Signatures

                
Architectural layers and exact code signatures across Go, Rust, C#, Java, Kotlin, Swift, Python, JavaScript, TypeScript, PHP, Ruby, Lua, C, and C++.

            
            
                RFC-ACP-010
                
Developer Experience & Migration Guide

                
Interactive CLI (`acp`), visual workflow studio, step debugger (`acp-test`), code generator (`acp codegen`), and migration paths from MCP, LangGraph & CrewAI.
